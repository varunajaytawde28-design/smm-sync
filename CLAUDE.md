# CLAUDE.md — smm-sync

> Generated by smm-sync. Edit smm.toml, then run `smm compile`. Do not edit this file directly.

## What is this project?

Coordination layer for simultaneous AI agents working on the same codebase. Automatically compiles shared context (CLAUDE.md, .cursorrules, AGENTS.md) from a single source of truth (smm.toml) and coordinates file ownership using Tuple Space protocol.

**Stack:** python 3.11+, click, jinja2, tomllib (stdlib), filelock

---

## Architectural Decisions

> These are immutable. Human-approved only. Read these before writing any code.

<details>
<summary><strong>os.rename() for file claiming</strong></summary>

**Why:** POSIX-atomic on both macOS and Linux. No external dependency, no daemon required, works in the same filesystem namespace.

</details>
<details>
<summary><strong>CRDT LWW-Register for state.json</strong></summary>

**Why:** Multiple agents writing simultaneously need merge-safe state. LWW is the simplest CRDT that handles the 'last writer wins' semantic of task claiming.

</details>
<details>
<summary><strong>Jinja2 templates for output files</strong></summary>

**Why:** Separates structure (smm.toml) from presentation (CLAUDE.md format). Templates can evolve without touching config schema.

</details>
<details>
<summary><strong>tomllib (Python 3.11 stdlib)</strong></summary>

**Why:** No external TOML dependency. TOML chosen over YAML for pyproject.toml familiarity and no indentation-sensitivity.

</details>
<details>
<summary><strong>Three-layer architecture: Insight / Pheromone / Interaction</strong></summary>

**Why:** Mirrors ant colony stigmergy: persistent rules (arch decisions), active signals (current locks), and history (RAG-accessible logs).

</details>

## Known Constraints

- Python 3.11+ only — tomllib is stdlib in 3.11+
- No network calls in Weekend 1 scope
- coordinator.py must work on macOS and Linux (POSIX-atomic os.rename)
- state.json must be human-readable JSON, never binary
- watcher.py and drift.py are STUBS ONLY — raise NotImplementedError

---

## Danger Zones

> Do not touch these without reading why first.

- Do not use os.rename() across filesystems — only atomic within the same filesystem
- state.json LWW merge uses wall clock — clock skew between machines can cause wrong winner
- Templates in src/smm_sync/templates/ are package data — must be listed in pyproject.toml package-data or they will be missing from installs
- Do not write derived or computed fields into smm.toml — it is the source of truth, not a cache
